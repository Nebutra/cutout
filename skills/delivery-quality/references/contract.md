# Delivery Quality Contract

## Status

- Capability status: `provider-required`
- Agent operations: `coding.review`, `coding.repair`
- MCP tools: `cutout_plan_coding_task`, `cutout_apply_coding_task`

## Requirements

- Acceptance criteria
- Controlled repair backend
- Reproducible harness

## Produced Evidence

- Quality report
- Viewport evidence
- Repair receipts
- Promotion decision

## Limitations

- Narrow tests do not prove visual completion
- Repairs stay scoped
- Promotion requires rebuilt evidence

## Invariants

- Pin the current Design IR revision.
- Preview before approved apply.
- Keep credentials host-owned.
- Claim only authoritative evidence.
